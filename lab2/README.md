# Лабораторная работа 2

Здесь оставлено только то, что нужно для задания:

- разделение подготовленного датасета на обучающую и тестовую выборки;
- решение задачи регрессии;
- оценка регрессионной модели;
- решение задачи классификации;
- оценка классификационной модели.

Для работы используются файлы из корня проекта:

- `processed_train.csv`
- `processed_valid.csv`

Эти файлы были получены в первой лабораторной работе, поэтому именно их и требует использовать методичка.

## Что находится в папке

- `main.py` — весь код лабораторной.
- `regression_plot.png` — сравнение линейной и полиномиальной регрессии.
- `confusion_matrix.png` — матрица ошибок для классификации.

## Что делает код

### 1. Подготовка данных

Скрипт объединяет `processed_train.csv` и `processed_valid.csv` в один подготовленный датасет.  
После этого булевы признаки переводятся в `0` и `1`, чтобы модели из `scikit-learn` работали без проблем.

### 2. Регрессия

Для регрессии выбран непрерывный признак `FoodCourt`.

Используются две модели:

1. `LinearRegression` — базовая модель по методичке.
2. `PolynomialFeatures(degree=2) + LinearRegression` — улучшение в рамках методички.

Почему выбран именно `FoodCourt`:

- это непрерывный признак;
- для него получилось наиболее адекватное качество среди простых вариантов;
- на нём хорошо видно, что полиномиальная модель лучше линейной.

### 3. Классификация

Для классификации используется целевой признак `Transported`.

Используются две версии одной и той же идеи:

1. базовая `LogisticRegression`;
2. улучшенная `PolynomialFeatures(degree=2) + LogisticRegression`.

То есть сама модель остаётся логистической регрессией, но перед ней добавляются полиномиальные признаки. Это позволило немного улучшить качество, не делая решение слишком сложным.

## Итоговые результаты

### Регрессия для `FoodCourt`

Линейная регрессия:

- `MSE = 0.5609`
- `RMSE = 0.7489`
- `MAE = 0.4106`
- `R2 = 0.2020`

Полиномиальная регрессия:

- `MSE = 0.4837`
- `RMSE = 0.6955`
- `MAE = 0.3092`
- `R2 = 0.3117`

Вывод:

- полиномиальная регрессия лучше линейной по всем метрикам;
- качество стало лучше, но `R2` всё ещё не очень высокий;
- значит зависимость есть, но она не описывается простой прямой.

### Классификация для `Transported`

Базовая логистическая регрессия:

- `Accuracy = 0.7838`
- `Precision = 0.7796`
- `Recall = 0.7957`
- `F1 = 0.7876`

Улучшенная логистическая регрессия с полиномиальными признаками:

- `Accuracy = 0.7918`
- `Precision = 0.7868`
- `Recall = 0.8048`
- `F1 = 0.7957`

Матрица ошибок для улучшенной модели:

- `[[672, 191], [171, 705]]`

Вывод:

- улучшенная модель немного лучше базовой;
- метрики `Accuracy`, `Precision`, `Recall` и `F1` выросли;
- итоговая классификационная модель работает достаточно хорошо.
